I've been using the BMAD method and, while it's definitely an improvement in regards to context management, I've had numerous issues with it.

- not enough research is typically done during the planning phase
- additional research is cumbersome because it requires using the elicitation menu multiple times. Thourough research should happen automatically
- The BMAD method relies on one, full stack developer agent for development instead of using more focused agents like the ones found in this repo. This pattern is repeated as well for other roles in the BMAD method
- BMAD offers an orchestration agent, but it is diffuctlt to know when to use it. Even when I do try, it doesn't appear to be very helpful

With all of these points in mind, I do understand why having so many agents can be detrimental as it requires tools like claude code to load all of the agents into memory. However, I do believe that the agent-based approach is the right one. I just need help figuring out how to make it work in practice.

My reasoning for this is that even though BMAD is a great tool in theory, I have yet to get it to develop a single application from start to finish. The reason is that developer agents are not very good at following through on story requirements. When met with challenges, they often attempt to "move the goal post" by taking it upon themselves to adjust the requirements, which in the end, results in broken code that doesn't meet the original requirements and needs to be fixed later. I've often seen agents try to work through a problem and ultimately go with a bare minimum approach that fails to address the root of the problem or satisfy all requirements. Instead, these agents should do what humans do. When met with challenges, they should reach out and ask for help from colleagues or friends (other agents in this case), perform additional research using tools such as context7 mcp or web search agents as well as leverage other tools at their disposal such as the sequential thinking mcp tools to work through problems and attempt to come up with a solution if one isn't readily available as human programmers must do. This process should also be automated as much as possible to save time and effort, requiring minimal human intervention.

When developing applications, AI agents should focus on test driven development as this is a much better way to develop software. Instead of writing code and then testing it, tests should be written first and the code should be written to pass the tests. This approach helps to ensure that the code is of high quality and meets the requirements. In addition, tests should be written at multiple levels including unit tests, integration tests, and end to end tests. This helps to ensure that the code is robust and reliable. Before development begins, multiple agents such as the scrum master, product owner and architect should verify that these tests meet the requirements defined and make any adjustments necessary. Lastly, the agent that will actually be responsible for writing the code should be involved in the test review process to ensure that the tests are easy to understand and that the implementation plan is feasible. This should also be automated as much as possible.
